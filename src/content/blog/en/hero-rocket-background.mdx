The homepage hero in [Astro Rocket](https://astrorocket.dev) has a large rocket icon sitting quietly in the background in dark mode. It fades in on page load, then floats slowly up and down in a continuous loop. It is low-opacity, centered behind the content, and colored with the active brand color — so it adapts automatically when you switch themes. It appears on desktop at all orientations and on mobile when the screen is in portrait mode. In light mode it is never shown.

## How it works

The rocket is a plain inline SVG rendered inside the `Hero` component. It is absolutely positioned to fill the hero section, centred both horizontally and vertically, and placed below all content with `pointer-events-none` so it never interferes with clicks or text selection.

The visibility is controlled entirely with Tailwind utility classes on the wrapper:

```html
<div class="pointer-events-none absolute inset-0 hidden dark:portrait:flex dark:lg:flex items-center justify-center">
```

- `hidden` — invisible by default on all screen sizes and orientations
- `dark:portrait:flex` — visible on portrait-oriented screens (typically phones held upright) when dark mode is active
- `dark:lg:flex` — visible at the `lg` breakpoint (1024 px) or wider when dark mode is active

The two dark-mode rules are additive: a tablet held in landscape at `lg` width satisfies `dark:lg:flex`; a phone held upright satisfies `dark:portrait:flex`. Neither rule fires in light mode. No JavaScript is involved.

## The SVG

The icon is the Lucide rocket, rendered with `fill="none"` and `stroke="currentColor"` so it inherits its color from the text color of its parent:

```html
<svg
  viewBox="0 0 24 24"
  fill="none"
  stroke="currentColor"
  stroke-width="0.75"
  stroke-linecap="round"
  stroke-linejoin="round"
  class="rocket-bg w-[min(55vw,420px)] portrait:w-[min(80vw,420px)] h-[min(55vw,420px)] portrait:h-[min(80vw,420px)] text-brand-500"
  style="opacity: 0;"
>
  <path d="M4.5 16.5c-1.5 1.26-2 5-2 5s3.74-.5 5-2c.71-.84.7-2.13-.09-2.91a2.18 2.18 0 0 0-2.91-.09z"/>
  <path d="m12 15-3-3a22 22 0 0 1 2-3.95A12.88 12.88 0 0 1 22 2c0 2.72-.78 7.5-6 11a22.35 22.35 0 0 1-4 2z"/>
  <path d="M9 12H4s.55-3.03 2-4c1.62-1.08 5 0 5 0"/>
  <path d="M12 15v5s3.03-.55 4-2c1.08-1.62 0-5 0-5"/>
</svg>
```

A few things worth noting:

- **`stroke-width="0.75"`** — thinner than the default Lucide stroke. At 420 px the paths are large, so a thinner stroke keeps the icon from looking heavy.
- **`w-[min(55vw,420px)]`** — the base size: scales with the viewport up to a maximum of 420 px on desktop.
- **`portrait:w-[min(80vw,420px)]`** — on portrait screens the icon grows to 80 vw (still capped at 420 px). A phone in portrait mode is narrow, so the larger fraction fills the background more convincingly without exceeding the cap.
- **`style="opacity: 0;"`** — the element starts fully transparent so there is no flash before the fade-in animation fires. The final opacity (0.27) is applied by the CSS animation, not a Tailwind class.

## Animation

The rocket runs two CSS animations in sequence, applied through the `rocket-bg` class.

**Phase 1 — fade in.** On load the rocket rises 12 px and fades from transparent to its resting opacity of 0.27 over 0.9 s. The easing is `cubic-bezier(0, 0, 0.2, 1)` — a quick start that decelerates sharply at the end, giving the entrance a sense of weight settling into place. The animation is delayed by 0.4 s so it begins after the hero content has already started sliding in.

```css
@keyframes rocket-fade-in {
  from { opacity: 0; transform: translateY(12px); }
  to   { opacity: 0.27; transform: translateY(0); }
}
```

**Phase 2 — float.** Once the fade-in is complete the rocket enters a continuous gentle float: it drifts 10 px upward over the first half of a 4.5 s cycle, then drifts back down over the second half. The loop starts at 1.3 s — just after the fade-in settles — so the two phases flow together without overlap.

```css
@keyframes rocket-float {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-10px); }
}
```

Both animations are applied together on the element:

```css
.rocket-bg {
  animation:
    rocket-fade-in 0.9s cubic-bezier(0, 0, 0.2, 1) 0.4s forwards,
    rocket-float 4.5s ease-in-out 1.3s infinite;
}
```

**Reduced motion.** Users who have enabled the "reduce motion" system preference get neither animation. The opacity snaps immediately to 0.27 so the rocket is still visible — just stationary.

```css
@media (prefers-reduced-motion: reduce) {
  .rocket-bg { animation: none; opacity: 0.27; }
}
```

## Color follows the theme automatically

The SVG uses `stroke="currentColor"`, which inherits from the `text-brand-500` class on the element. `brand-500` is a CSS custom property — not a fixed hex value. When the user switches from blue to violet, orange, or any of the other available themes, the property updates and the rocket stroke color updates with it. No extra code is needed. See [How Astro Rocket's Design System Works](/blog/design-system-color-tokens) for a full walkthrough of every colour token.

## The prop

The feature is controlled by a `showRocketBg` boolean prop on the `Hero` component. The default is `false`, so adding the prop enables it:

```astro
<Hero layout="centered" size="xl" showRocketBg showScrollIndicator ...>
```

## How to turn it off

Remove the `showRocketBg` prop from the `<Hero>` call in `src/pages/index.astro`:

```astro
<!-- Before -->
<Hero layout="centered" size="xl" class="hero-dark-gradient" showScrollIndicator showRocketBg descriptionClass="max-w-3xl">

<!-- After -->
<Hero layout="centered" size="xl" class="hero-dark-gradient" showScrollIndicator descriptionClass="max-w-3xl">
```

One word removed, feature gone. No CSS to clean up, no variables to reset.

## Why portrait mobile and dark mode only

On landscape mobile and small tablets the screen is wide but short, so a large centered icon would compete directly with the hero text and buttons. Portrait mode gives the rocket enough vertical room to sit comfortably behind the content without crowding it.

Light mode was considered but the icon at 0.27 opacity on a light background reads more as clutter than atmosphere. The dark hero gradient — brand colour fading to black — gives the rocket exactly the right contrast to feel intentional. See [Hero Gradient — Brand to Black, Dark Mode Only](/blog/dark-mode-hero-gradient) for how that gradient is set up.
